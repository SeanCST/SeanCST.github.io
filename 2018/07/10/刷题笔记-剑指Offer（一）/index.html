刷题笔记 - 剑指 Offer（一）

发表于 2018-07-10

第一篇博客

三月底研究生复试完就搭好了博客，心想着一个月能写一篇文章，但是由于个人属于拖延症晚期患者加上不知道写些什么，所以又拖延了好几个月。这周由于刚离职，空闲时间要多一些，故决心开始写下第一篇博客，其实这并不算是什么文章，只是记录一下自己的刷题过程罢了。不过应该至少也有以下几个好处：

  • 熟悉 / 回忆 Markownd 语法 ：自 16 年用 Markdown 写过几篇读书笔记发表在简书之后，再未用过 Markdown，现在已经忘得差不多了，需要再捡起来
  • 养成写博客的习惯 ：既然当下不知道该写什么文章，也不够水平写一些高水平的文章，那从记录刷题做起也是一个不错的选择，至少可以持续一个输出的过程，养成一个好的习惯
  • 记录刷题心得 促进自己刷题 ：从大三开始有刷 LeetCode 一些 Easy 的问题，到后来大四看了《剑指 Offer》这一本比较经典的算法刷题书籍，不过由于时间比较紧，大部分题只是看了一两遍或纸上写了一些，没有亲手编译运行提交，之前刚好看到 牛客网 上有这个题库，所以再在线上刷一遍。准备在刷完《剑指 Offer》题库之后继续刷 LeetCode 或 PAT 甲级 / 顶级，并用博客记录下这一过程，当然不可能每题都记录，应该只是挑选记录一些比较经典或者自己刷题过程中印象比较深的题。现在的目标是每天刷一题，修炼基本功，慢慢地提高自己对这些经典的数据结构和算法问题的理解和解决能力。

题目 - 左旋转字符串

题目描述

汇编语言中有一种移位指令叫做循环左移（ROL），现在有个简单的任务，就是用字符串模拟这个指令的运算结果。对于一个给定的字符序列S，请你把其循环左移K位后的序列输出。例如，字符序列S=”abcXYZdef”,要求输出循环左移3位后的结果，即“XYZdefabc”。是不是很简单？OK，搞定它！

这道题一眼看上去就很简单，很容易就能想到把字符串分为前后两个字符串 s1 和 s2，前一个字符串 s1 长度为n， 后一个字符串长度为str.length() - n，将字符串由s1 + s2转化为s2 + s1就可以得到结果。只要稍微了解一点 STL 就可以很快写出以下代码完成提交并通过：

但是上面这种方法并不是这道题真正要考察的内容，显然存在着更简洁有效的解决方法，比如将字符串后面再重复一遍，截取其中相应的部分，如：S = "abcdXYZ"， 将其重复一遍为SS = "abcdXYZabcdXYZ"，如 n 为 4, 则截取 SS 内从下标 4 开始的 S.length() 个字符，得到结果"XYZabcd"。这样的解决方法就要比上面的方法灵活许多了。

而本题真正考察的方法十分巧妙，是要利用 C++ 的reverse函数，三次调用，便得到最终答案：

  • 第一次调用是翻转整个字符串，如S = "abcdXYZ"，令S = s1 + s2，其中 s1 = "abcd"， s2 = "XYZ"，则翻转之后整个字符串前后颠倒了，变成了S = "ZYXdcba"
  • 第二次调用是翻转此时的字符串头直到str.length() - n处，如 n 为 4，则变为S = "XYZdcba"，将一开始位于后面的一部分字符串（即上面的 s2）放在了正确的位置
  • 第三次调用是翻转字符串str.length() - n处直到末尾的这一部分，这一段的长度为 n，结果变为S = "XYZabcd"，将一开始位于前面的一部分字符串（即上面的 s1）放在了正确的位置，这样便得到了最终的正确答案
