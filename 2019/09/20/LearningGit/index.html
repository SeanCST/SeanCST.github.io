Learning Git - 常用 Git 命令总结

发表于 2019-09-20

说明

本文档内的命令及说明是学习 廖雪峰老师的 Git 课程 时所做总结，以方便忘记命令时查看。

一、创建版本库

  • git init ： 把当前目录变成Git可以管理的仓库
  • git add <file>： 把文件添加到仓库 （把要提交的所有修改放到暂存区（Stage））
  • git commit -m <message>： 把文件提交到仓库，-m 后面输入的是本次提交的说明（一次性把暂存区的所有修改提交到分支）

二、管理修改

  • git status：时刻掌握仓库当前的状态
  • git checkout -- <file>：丢弃工作区的修改，回到最近一次git commit或git add时的状态，没有--，就变成了“切换到另一个分支”的命令
  • git diff <file>：查看更改
  • git log：显示从最近到最远的提交日志
    • git log --pretty=oneline：每条记录以一行显示
    • git log --pretty=oneline --abbrev-commit
  • git reset --hard commit_id：回退到某一个版本
    • HEAD^：HEAD 指向的版本就是当前版本，回退到上 n 个版本，n 为^的数目，若 ^ 数目太多则用 ~n 代替
    • commit_id：直接回退到版本号
  • git reset HEAD <file>：添加到了暂存区时，想丢弃修改
  • git reflog：记录提交或回退的每一次记录
  • git diff HEAD -- <file>：查看工作区和版本库里面最新版本的区别
  • git rm <file>：用于删除一个文件
    • git checkout -- <filename>：误删了本地文件，用版本库里的文件恢复本地文件

三、远程仓库

  • 先有本地库，后有远程库

    • git remote add origin git@server-name:path/repo-name.git：将本地的仓库关联到远程仓库
    • git push -u origin master：第一次推送 master 分支的所有内容

  • 先有远程

    • git clone：将远程仓库克隆到本地

  • git push origin master：推送最新修改

四、分支管理

  • git branch：查看分支
  • git branch <name>：创建分支
  • git checkout <name>：切换分支
  • git checkout -b <name>：创建 + 切换分支
  • git merge <name>：合并某分支到当前分支
  • git branch -d <name>：删除分支
  • git log --graph --pretty=oneline --abbrev-commit：查看简单的 log 图示
  • 贮存/恢复
    • git stash ：贮存现场
    • git stash list：查看贮存的列表
    • git stash apply：恢复
    • git stash apply stash@{0}：恢复指定的贮存
    • git stash drop：删除贮存
    • git stash pop：恢复并删除贮存
  • git merge --no-ff -m "merge with no-ff" dev：准备合并 dev 分支，注意 --no-ff 参数，表示禁用 Fast forward，Git 就会在 merge 时生成一个新的 commit，这样，从分支历史上就可以看出分支信息
  • git cherry-pick <commit ID> ：把其他分支上提交的修改“复制”到当前分支
  • git branch -D <name>：丢弃一个没有被合并过的分支
  • git remote -v：查看远程库的信息
  • git push origin <branch-name>：推送分支
  • git checkout -b <branch-name> origin/<branch-name>：在本地创建和远程分支对应的分支
  • git branch --set-upstream-to=origin/<branch-name> <branch-name>：设置 dev 和 origin/dev 的链接（如果 git pull 提示 no tracking information，则说明本地分支和远程分支的链接关系没有创建）
  • git rebase
    • rebase 操作可以把本地未 push 的分叉提交历史整理成直线
    • rebase 的目的是使得我们在查看历史提交的变化时更容易，因为分叉的提交需要三方对比

五、标签管理

  • git tag <tagname>：新建一个标签，默认为 HEAD，也可以指定一个 commit id，eg: git tag v0.9 f52c633
  • git tag -a <tagname> -m "blablabla..."：指定标签信息
  • git tag：查看所有标签
  • git show <tagname>：看到说明文字
  • git push origin <tagname>：推送本地标签
  • git push origin --tags：推送全部未推送过的本地标签
  • git tag -d <tagname>：删除一个本地标签
  • git push origin :refs/tags/<tagname>：删除一个远程标签

六、自定义 Git

  • git config --global alias.st status：给 git 命令设置别名
  • 当前仓库的配置在 ./.git/config 中
  • 全局配置在 ~/.gitconfig 中
